qsTr, QTranslator, and lupdate/lrelease Workflows

Marking Strings for Translation

The qsTr() function marks a string as translatable. It takes an optional disambiguation comment and plural-form count:

// Basic translation
Text { text: qsTr("File") }

// With disambiguation (same source string, different context)
Text { text: qsTr("Edit", "menu item") }
Text { text: qsTr("Edit", "button label") }

// Plural forms
Text { text: qsTr("%n item(s) selected", "", selectionCount) }

The string argument to qsTr() is the source string — the English (or base language) text that lupdate extracts. At runtime, Qt looks up the translation for this string in the loaded .qm file.

qsTrId(): For applications that separate translation IDs from source strings (common in enterprise workflows):

Text { text: qsTrId("file-menu-open-action") }

qsTranslate(): Translates a string from a specific context, rather than the current QML file’s context:

Text { text: qsTranslate("MainMenu", "File") }

The lupdate / lrelease Workflow

lupdate scans source files and extracts translatable strings into .ts (Translation Source) XML files. lrelease compiles .ts files into binary .qm (Qt Message) files that Qt loads at runtime.

Step 1: Create or update .ts files

# lupdate all QML files in the project
lupdate qml/ -ts translations/myapp_de.ts translations/myapp_ja.ts

# Or with a .pro file
lupdate myapp.pro

Step 2: Translate

Send .ts files to translators. They use Qt Linguist (linguist) to enter translations:

<!-- translations/myapp_de.ts (excerpt) -->
<TS version="2.1" language="de_DE">
    <context>
        <name>Main</name>
        <message>
            <source>File</source>
            <translation>Datei</translation>
        </message>
        <message>
            <source>%n item(s) selected</source>
            <numerusform>%n Element ausgewählt</numerusform>
            <numerusform>%n Elemente ausgewählt</numerusform>
        </message>
    </context>
</TS>

Step 3: Compile

lrelease translations/myapp_de.ts -qm translations/myapp_de.qm

Step 4: Load at runtime

# Python
from PySide6.QtCore import QTranslator, QLocale, QLibraryInfo

translator = QTranslator(app)
locale = QLocale.system()
if translator.load(locale, "myapp", "_", ":/translations"):
    app.installTranslator(translator)

// C++
QTranslator translator;
const QLocale locale = QLocale::system();
if (translator.load(locale, "myapp", "_", ":/translations"))
    app.installTranslator(&translator);

CMake Integration

qt_add_translations(myapp
    TS_FILES
        translations/myapp_de.ts
        translations/myapp_fr.ts
        translations/myapp_ja.ts
    QM_FILES_OUTPUT_VARIABLE qm_files
)

qt_add_resources(myapp "translations"
    PREFIX "/translations"
    BASE "${CMAKE_CURRENT_BINARY_DIR}"
    FILES ${qm_files}
)

qt_add_translations runs lupdate during the build and ensures .qm files are produced and embedded as resources.

Runtime Language Switching

Switching language at runtime requires reinstalling the translator and signaling QML to retranslate:

void App::setLanguage(const QString &languageCode) {
    app->removeTranslator(&m_translator);
    if (m_translator.load(":/translations/myapp_" + languageCode + ".qm")) {
        app->installTranslator(&m_translator);
    }
    // Signal QML to re-evaluate all qsTr() calls
    engine->retranslate();
}

QQmlEngine::retranslate() (Qt 5.10+) triggers re-evaluation of all qsTr() binding expressions in all loaded QML components. UI updates immediately without a restart.

Accessibility Tree and Accessible Attached Properties

Qt Quick’s accessibility support exposes UI elements to assistive technologies (screen readers, switch access, magnifiers) through the platform’s accessibility API (AT-SPI on Linux, UIAutomation on Windows, NSAccessibility on macOS).

The Accessible Attached Type

Every Item can declare its accessibility role and properties via the Accessible attached type:

Rectangle {
    id: customButton

    Accessible.role: Accessible.Button
    Accessible.name: buttonLabel.text
    Accessible.description: "Saves the current document"
    Accessible.onPressAction: clicked()

    // Enable focus for keyboard navigation
    focusPolicy: Qt.TabFocus
    Keys.onReturnPressed: clicked()
    Keys.onSpacePressed: clicked()

    signal clicked()

    Text { id: buttonLabel; text: "Save" }
    TapHandler { onTapped: customButton.clicked() }
}

Accessible.role: Accessible.Slider
Accessible.name: "Volume"               // label read by screen reader

Qt Quick Controls Accessibility

Qt Quick Controls 2 controls include built-in Accessible declarations. Button, TextField, CheckBox, Slider etc. are all accessible by default — their Accessible.role, Accessible.name, and state properties are set automatically. When building custom controls from scratch, you must add Accessible declarations explicitly.

Testing Accessibility

On Linux, use accerciser (AT-SPI inspector) to inspect the accessibility tree. On macOS, use the Accessibility Inspector (in Xcode’s Additional Tools). On Windows, use Inspect.exe (from the Windows SDK).

Design Token Systems and Runtime Theme Switching

Design Tokens

Design tokens are named values — colors, spacing, typography — that separate visual style decisions from component implementation. In a QML application, tokens are typically exposed as a singleton:

// Theme.qml (singleton)
pragma Singleton
import QtQuick

QtObject {
    readonly property color primary: "#3daee9"
    readonly property color primaryVariant: "#1a6e9a"
    readonly property color secondary: "#27ae60"
    readonly property color background: "#1e1e2e"
    readonly property color surface: "#313244"
    readonly property color onBackground: "#cdd6f4"
    readonly property color onSurface: "#cdd6f4"
    readonly property color error: "#f38ba8"

    readonly property int radiusSmall: 4
    readonly property int radiusMedium: 8
    readonly property int radiusLarge: 16

    readonly property int spacingXS: 4
    readonly property int spacingS: 8
    readonly property int spacingM: 16
    readonly property int spacingL: 24
    readonly property int spacingXL: 40

    readonly property font bodyFont: Qt.font({ family: "Inter", pixelSize: 14 })
    readonly property font headingFont: Qt.font({ family: "Inter", pixelSize: 20, weight: Font.DemiBold })
}

Components use token names rather than hardcoded values:

import com.example.app

Rectangle {
    color: Theme.surface
    radius: Theme.radiusMedium
    border.color: Theme.primary
    border.width: 2

    Text {
        font: Theme.bodyFont
        color: Theme.onSurface
    }
}

Runtime Theme Switching

For switchable light/dark or branded themes, make the Theme singleton mutable:

// Theme.qml
pragma Singleton
import QtQuick

QtObject {
    id: theme

    property string current: "dark"

    readonly property color primary: current === "dark" ? "#3daee9" : "#1565c0"
    readonly property color background: current === "dark" ? "#1e1e2e" : "#eff1f5"
    readonly property color onBackground: current === "dark" ? "#cdd6f4" : "#4c4f69"
    // ...
}

When Theme.current changes, all bindings that reference Theme.primary, Theme.background, etc. re-evaluate automatically via the binding engine. The entire UI repaints with the new theme values in one frame.

Integrating with Qt’s Palette System

Qt Quick Controls 2’s palette system (Chapter 4) and custom design tokens can coexist. Use palette properties for controls that derive from Control, and token properties for custom items:

ApplicationWindow {
    // Set palette for all Controls
    palette.button: Theme.primary
    palette.buttonText: Theme.onPrimary
    palette.highlight: Theme.secondary

    // Custom items use Theme directly
    MyCustomHeader {
        background: Theme.surface
    }
}

This hybrid approach ensures built-in controls adopt theme colors without overriding them component-by-component, while custom items have full token flexibility.

Animated Theme Transitions

Smooth theme switching requires animating the color change across the entire UI. With bindings to Theme.background etc., a Behavior on color at the root level animates all changes:

// Apply to all Rectangles via a base component
Rectangle {
    Behavior on color {
        ColorAnimation { duration: 200; easing.type: Easing.OutCubic }
    }
}

For a more targeted approach, animate the Theme.current switch with a opacity-based crossfade:

Item {
    Loader { id: currentThemeView; source: "MainContent.qml"; opacity: 1 }
    Loader { id: nextThemeView; opacity: 0 }

    function switchTheme(newTheme) {
        nextThemeView.source = "MainContent.qml"
        // ... crossfade animation ...
        Theme.current = newTheme
    }
}

Summary

Internationalization in Qt Quick is built around qsTr() for string marking, lupdate/lrelease for the translation compilation pipeline, and QQmlEngine::retranslate() for live language switching. Accessibility requires explicit Accessible attached property declarations on custom items; Qt Quick Controls handle this automatically for built-in controls. Theming through design tokens creates a single source of truth for visual values, enabling runtime theme switching through QML’s binding engine with zero imperative update code. Combining these three systems produces applications that are globally deployable, universally accessible, and visually adaptable without architectural changes.
